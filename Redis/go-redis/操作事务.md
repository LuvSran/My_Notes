## TxPipeline
- **`DB.TxPipeline()`**: 初始化一个本地缓冲区。
- **`tx.Set(...)` / `tx.Get(...)`**: 仅仅是将命令记录在本地缓冲区，不发送给 Redis
- **`tx.Exec()`**:它会一次性通过网络发送以下报文：
    > `MULTI` + `Get`，`Set`等语句+ `EXEC`
```go
//TxPipeline()
func main() {  
    DB := redis.NewClient(&redis.Options{  
       Addr:     "localhost:6379",  
       DB:       0,  
       Password: "",  
    })  
    tx := DB.TxPipeline()  
    
    tx.Set("name", "momo", 0)  
    val := tx.Get("name")  
    
    _,err:=tx.Exec()
    if err!=nil{
	    fmt.Println(err)
	    return
    }  
    fmt.Println(val.Val())  
}
```
## TxPipelined
TxPipelined 会自动在函数开始时发送 MULTI，在结束时发送 EXEC
```go
//TxPipelined()
func main() {  
    DB := redis.NewClient(&redis.Options{  
       Addr:     "localhost:6379",  
       DB:       0,  
       Password: "",  
    })   
    
    _, err := DB.TxPipelined(func(pipe redis.Pipeliner) error {  
    
       pipe.Set("name", "momo", 0)  
       val = pipe.Get("name")  
       // 如果return的不是nil，事务会被取消(不执行EXEC)  
       return nil  
    })  
    
    if err != nil {  
       fmt.Println(err)  
    }  
    fmt.Println(val.Val())  
}
```
