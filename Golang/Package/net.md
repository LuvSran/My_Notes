# net.Coon
### `net.Conn` 的核心作用和特性

 双向通信管道
`net.Conn` 是一个抽象概念，它必须能够**发送数据**（写入）和**接收数据**（读取）。它是一个全双工（Full-duplex）的通道，意味着客户端和服务器可以同时发送和接收数据。

|**方法**|**作用**|**IM 项目中的应用**|
|---|---|---|
|**`Read(b []byte) (n int, err error)`**|**从连接中读取数据**。它会尝试将连接对端发送的数据填充到切片 `b` 中。这是一个阻塞操作。|用于接收客户端（用户）发送过来的聊天消息、心跳包等。|
|**`Write(b []byte) (n int, err error)`**|**向连接中写入数据**。它将切片 `b` 中的数据发送给连接对端。|用于服务器向客户端推送消息、发送确认信息等。|
|`Close() error`|**关闭连接**。释放连接占用的系统资源。一旦关闭，后续的 Read/Write 操作将失败。|当用户断开连接时，或者服务器主动断开连接时调用（通常使用 `defer`）。|
|`LocalAddr() net.Addr`|返回连接的**本地网络地址**（服务器的 IP 和端口）。|用于日志记录或调试，了解是哪个本地接口处理了连接。|
|`RemoteAddr() net.Addr`|返回连接的**对端网络地址**（客户端的 IP 和端口）。|用于识别是哪个客户端用户建立了连接。|
# nat.Dail
用于**主动发起网络连接**（客户端行为），建立到远程服务器的 TCP、UDP、Unix 域套接字等连接。
函数签名
```go
func Dial(network, address string) (Conn, error)
```
- **`network`**：指定网络协议类型（字符串）
- **`address`**：目标服务器的地址（`"host:port"` 格式）
- **返回值**：
    - `net.Conn`：一个通用的连接接口，可用于读写数据
    - `error`：连接失败时返回错误
    
|`network`|说明|
|---|---|
|`"tcp"`|TCP 连接（最常用）|
|`"tcp4"`|仅 IPv4 的 TCP|
|`"tcp6"`|仅 IPv6 的 TCP|
|`"udp"`|UDP 连接（无连接，但 Dial 会“伪连接”以便后续 Write）|
|`"unix"`|Unix 域套接字（本地进程通信）|
