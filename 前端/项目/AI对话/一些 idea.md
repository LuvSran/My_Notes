数据结构与算法优化：

1. Trie / Radix Tree  
    用途：指令补全、敏感词前缀匹配、命令路由。  
    亮点：高性能前缀检索，比线性扫描更工程化。
    
2. Aho-Corasick + 动态词库增量构建  
    用途：输入输出审核。  
    亮点：支持热更新词库（分片重建+原子切换），可讲并发与可用性。
    
3. MinHash + LSH  
    用途：对话去重、相似问题缓存命中。  
    亮点：降低模型调用成本，面试很加分。
    
4. LRU/LFU + TTL 缓存（哈希表 + 双向链表）  
    用途：prompt->response 缓存、embedding 缓存。  
    亮点：典型数据结构工程化应用，直接体现性能优化能力。
    
5. Bloom Filter / Cuckoo Filter  
    用途：快速判重（是否见过某条文本/片段）。  
    亮点：节省内存，适合大规模日志或消息流预过滤。
    
6. 滑动窗口 + 单调队列  
    用途：限流（每用户每分钟请求数）、异常流量检测。  
    亮点：稳态服务能力，和后端网关场景强相关。
    
7. Top-K（堆）+ 倒排索引  
    用途：轻量检索（关键词召回）和热门问题统计。  
    亮点：可作为“RAG 前置版”，实现成本低。
    
8. 编辑距离（Levenshtein）+ BK-Tree  
    用途：拼写纠错、近似命令匹配、脏词变体识别。  
    亮点：提高鲁棒性，补 AC 的“精确匹配”短板。
    
9. 前缀和 / 差分思想做 token 预算控制  
    用途：会话裁剪时快速估算上下文预算。  
    亮点：把算法思维用于 LLM 工程细节。
    

10. 有限状态机（FSM）  
    用途：流式输出状态管理（正常/审核中/拦截/中止）。  
    亮点：把复杂分支变成可维护状态图。
    

最推荐你优先做的 3 个组合（性价比高）：

1. AC自动机 + BK-Tree（审核精准+模糊补强）
2. LRU缓存 + MinHash/LSH（降本提速）
3. 滑动窗口限流 + FSM流控（稳定性提升）
