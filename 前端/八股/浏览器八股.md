# 进程和线程
**进程**
进程是计算机中正在运行的程序的实例，一个进程就是一个程序运行实例。它有独立的地址空间，进程间资源是隔离的。进程通过（IPC）机制来交换数据和进行通信，方式包括管道、消息队列、共享内存。进程的切换开销较大，因为涉及页表切换。
**线程**
线程是进程的子任务，一个进程可以包含多个线程。同一个进程中的线程共享进程资源，但有私有的栈和寄存器。多个线程可以在同一进程内并发执行，他们的通信简单，可直接读写共享内存。线程切换开销小，因为不需要切换页表([[页表(计组)|定义]]),(仅需要切换寄存器和栈指针)
**区别**
* 进程是资源分配的最小单位，线程是 cpu 调度的最小单位
* 进程有独立的地址空间，资源相互隔离。线程是共享所属进程的资源的
* 进程切换(创建和销毁)开销大，线程开销小
* 进程通信复杂，需要 IPC 机制。线程通信简单，共享内存
# 浏览器渲染流程
1.  **解析 HTML → 构建 DOM 树**
* 浏览器从服务器收到 HTML 字节流后，会进行分词和构建树，逐步将 HTML 标签转换为 DOM 节点。
* DOM 树描述了页面的结构与节点关系，不包含样式
* 遇到 \<script>(没有 defer/async 属性) 会阻塞解析，直到 JS 执行完毕,因为 JS 可能会修改 DOM 结构
 
2. **解析 CSS → 构建 CSSOM 树**
* 解析所有 CSS（内联、外链、样式表），生成 CSSOM 样式对象模型
* CSSOM 包含所有样式规则、继承、层叠、优先级计算，是带样式的树形结构
*  CSS 同样会阻塞渲染，因为为渲染树需要结合 DOM 和 CSSOM

3. **合并 DOM + CSSOM → 生成渲染树**
* 只保留可见节点：剔除 display:none、head 内节点等不可见元素
*  为每个可见节点匹配最终计算样式（继承 + 层叠后的最终样式）
4. **布局（Layout / Reflow）→ 计算几何信息**
- 也叫重排，核心是计算元素的盒模型、位置、尺寸
- 依据渲染树，计算每个元素在视口中的坐标、宽高、排版（流式布局、Flex、Grid 等）
- 只要页面结构 / 尺寸变化，就会触发重新布局，开销极大
5. **绘制（Paint / Repaint）→ 填充像素**
- 也叫重绘，将布局后的元素填充实际像素
- 绘制内容：颜色、背景、边框、阴影、文字、图像等
- 绘制是分层进行的，不改变布局只改样式（如颜色）只会触发重绘，比重排轻量
6. **合成（Composite）→ 图层合并上屏**
- 浏览器将多个绘制好的图层，按层级关系合成最终画面，发送给 GPU 显示
- 只触发合成的操作（如 transform、opacity）不经过布局和绘制，开销最小
*进阶：*
- **重排（Reflow）**：当元素的几何属性（如 `width`, `top`, `margin`）发生变化，或者 DOM 结构改变时，浏览器需要重新计算布局。重排代价很大，且重排一定会触发重绘。
- **重绘（Repaint）**：当元素外观改变（如 `color`, `background-color`）但不影响布局时，只触发重绘。
- **合成（Composite）**：这是性能最优的。如果我们修改 `transform` 或 `opacity`，浏览器可以直接跳过重排和重绘，仅由合成线程和 GPU 处理，也就是硬件加速。
# 浏览器有哪些进程
- **主进程**：负责处理用户输入、渲染页面等主要任务。
- **渲染进程**：渲染进程负责解析`HTML`、`CSS`和`JavaScript`，并将网页渲染成可视化内容。
- **GPU进程**：负责处理浏览器中的`GPU`加速任务。
- **网络线程**：网络进程负责处理浏览器中的网络请求和响应，包括下载网页和资源等。
- **插件进程**：负责浏览器插件运行。
