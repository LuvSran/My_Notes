# GET 和 POST 的区别
#### GET
* 获取数据
* 查询, 搜索，获取资源
* 幂等，多次请求结果相同
*  请求可以被浏览器缓存，提升性能。所以静态资源（CSS、JS、图片）可以使用 GET 请求。
* 参数直接拼接在 URL 后面，直接暴露在浏览器地址栏中
* HTTP 协议本身对数据长度没有规定，但是一般浏览器对 URL 长度有限制

#### POST
* 提交数据，通常会更改服务器上的数据
* 创建，更新，提交表单，上传文件
* 非幂等，多次请求可能产生不同结果
* POST 请求默认不会被缓存，因为每次提交都可能改变服务器状态。刷新或后退时，浏览器会弹出警告，防止用户重复提交
* 参数放在请求体中，而不是暴露在 URL 中，相对 GET 更隐蔽。但其实都是明文传输，都不安全，只有在 HTTPS 协议下才是安全的。HTTPS 会对整个 HTTP 报文进行加密。
* 传输的数据长度理论上没有长度限制，实际上取决于服务器配置
